# 스토리라인 통합 — Problem → Solution 슬라이드 흐름 (v1)

> 목적: Gemini 초안(`WTIA_AI_Cost_Gov_Moat_Why_Now_v1.md`)의 논리를 팀 확정 포지셔닝(중간 시장 × COGS 원가 설명)에 맞춰 Problem·Solution 슬라이드 흐름에 녹인 대본.
> 확정 사항(2026-07-23): **① 리드 = 비용 절감 아님, 원가 귀속·설명(COGS). ② 절감(캐싱·라우팅)은 메인 서사가 아니라 '실행 전 통제를 가능케 하는 기술적 근거 + 부수 효과'로만 배치. ③ Moat = 6개월이면 복제되는 기능 갭이 아니라 고객 데이터 축적 기반(유닛 이코노믹스·동종 벤치마크).**
> 상태: 슬라이드 카피 초안. 인터뷰(0건) 검증 후 숫자 확정.

## 0. 흐름 한눈에 (Market Shift 구조)

```
Shift          지출 결정 주체가 사람 → 모델로 넘어갔다
  └ Villain    사후 대시보드·정적 키 — "불 난 뒤 CCTV"
      └ 증거    전 규모 실사고 ($4.2K 개발자 1인 ~ $500M 대기업)
          └ 진짜 통증   "얼마 나갈지 모른다"가 아니라 "왜 나갔는지 설명 못 한다"
              └ Promised Land   고객 1명당·기능별 AI 원가를 실시간으로 설명할 수 있는 회사
                  └ Vehicle(Solution)   실행 경로 위에서 원가를 귀속·통제하는 레이어
                      └ Moat   고객이 늘수록 정확해지는 유닛 이코노믹스·벤치마크 데이터
```

핵심 전환: 문제 슬라이드의 클라이맥스를 **"비용 급증(How much?)"이 아니라 "원가 설명 불가(Why?)"**에 둔다. 급증은 통증의 계기일 뿐이고, 상시 니즈이자 지불 이유는 "이사회·투자 실사·가격 책정에 낼 수 있는 원가 설명"이다.

## 1. PROBLEM 슬라이드 (slide 3~5)

### Slide 3 — The Shift (지출 결정권의 이동)

**헤드라인:** "AI 지출을 결정하는 주체가 사람에서 모델로 넘어갔다."

**본문 카피:**
- 전통적 소프트웨어에서 비용은 개발자가 코드에 명시한다 — 언제, 몇 번 API를 부를지 사람이 정한다. 예측 가능하고 통제 가능하다.
- 에이전틱 AI는 다르다. 하나의 사용자 요청을 처리하며 **모델이 스스로** 어떤 툴을, 언제, 몇 번 호출할지 판단한다. 지출의 방아쇠를 사람이 아니라 모델이 당긴다.
- 프로덕션 에이전트의 토큰 소모는 챗봇의 ~50배. 지출 결정권이 넘어간 순간, 재무는 통제권을 잃었다.

> Why Now 근거: Meta조차 사내 리더보드("Claudeonomics")로 사용량을 게임화하다 유휴 에이전트를 돌리는 "tokenmaxxing"이 발생, 비용이 연 수십억 달러에 이르자 리더보드를 폐기하고 자체 AI Gateway를 구축했다. 세계 최고 기술사가 "사용량 보상 → 통제"로 급선회한 것이 지금이 그 변곡점이라는 증거.

### Slide 4 — The Villain (사후 대시보드·정적 키)

**헤드라인:** "지금 도구들은 불이 난 뒤에 CCTV를 돌려본다."

**본문 카피:**
- 기존 관측 도구(대시보드·빌링 리포트)는 **월 단위·사후**다. 이미 나간 돈을 보여줄 뿐, 실행 중에 막지 못한다.
- 정적 API 키·고정 상한(max_iterations)은 무딘 칼이다. 정상 세션과 폭주 세션을 구분하지 못한 채 일괄로 자르거나, 아예 못 자른다.
- 그리고 결정적으로 — 이 도구들은 **LLM 호출 비용까지만** 본다. 에이전트가 자율로 부르는 검색·크롤링·서드파티 SaaS API는 비용 그림 밖에 흩어져 있다. **총원가를 아무도 한 화면에서 설명하지 못한다.**

> 경쟁 정직성(백업): LiteLLM·Portkey 등 LLM 게이트웨이는 LLM 예산·세션 상한이 이미 성숙하다. 우리가 공격하는 갭은 그 위 — **"기능이 있다"와 "조직이 원가를 예측·귀속·설명한다"는 다르다.** 게이트웨이를 깐 기업도 여전히 총원가를 설명하지 못한다.

### Slide 5 — 진짜 통증 (증거 + 재정의)

**헤드라인:** "문제는 '얼마 나갈지 모른다'가 아니다. '왜 나갔는지 설명 못 한다'는 것이다."

**증거 밴드(전 규모 실사고):**
- 개발자 1인, 무한 재시도 루프 → 11일간 인지 못한 채 $47K 청구 (why-now 사례)
- B2B SaaS, 하룻밤 API 사용량 400% 급증("Runaway API Usage")
- 대기업 규모에선 통제 실패가 연 수십억~수억 달러 단위로 커진다

**재정의 카피:**
- 급증은 통증의 *계기*일 뿐이다. 진짜 손실은, 사고가 지난 뒤에도 **"이 고객 한 명을 서비스하는 데 우리 AI 원가가 얼마인지, 어느 기능이 마진을 먹는지"를 답하지 못한다**는 데 있다.
- 이건 사고가 났을 때만 뜨거운 진통제 니즈가 아니다. **가격 책정·이사회 보고·투자 실사**에 매달 필요한 상시 니즈다. AI 원가는 이제 COGS(매출원가)이고, 설명 못 하는 COGS는 재무 리스크다.

> 지속 가능한 AI SaaS의 기준선: "LLM 비용 < COGS의 20%". 이 선을 아는지조차 대부분 모른다.

## 2. SOLUTION 슬라이드 (slide 6~8)

### Slide 6 — Promised Land → Vehicle

**헤드라인:** "고객 1명당·기능별 AI 원가를, 실시간으로 설명할 수 있는 회사가 된다."

**포지셔닝 한 줄:** "우리는 'AI 비용을 줄여드립니다'가 아니라 **'AI 원가를 설명할 수 있게 해드립니다'**를 판다."

**Vehicle 정의:**
- 사후 대시보드(경로 밖)가 아니라, **실행 경로 위에서** 에이전트가 유발하는 LLM + 툴 + 외부 API 호출을 요청 단계에서 귀속·통제하는 레이어.
- "불 난 뒤 CCTV가 아니라, 불붙기 전 차단기." — 실행 전에 막고, **왜 막았는지 감사 등급으로 설명한다.**
- 진입은 신뢰 문제를 피해 **경로 밖 리포트(프록시 없음)**로 시작 → 신뢰를 얻은 뒤 경로 안으로. (리스크 대책 반영)

### Slide 7 — 어떻게 작동하나 (기술적 근거, 정확한 용어)

> ⚠️ Gemini 초안의 "커널 수준", "C/C++ 메모리 관리를 라우팅에 접목" 같은 표현은 폐기. LLM 호출은 네트워크 바운드이고 "커널 캐싱"은 성립하지 않는다 — 기술 심사위원 앞에서 즉시 무너진다. 아래는 실제로 방어 가능한 용어로 재서술.

솔루션을 떠받치는 세 가지 메커니즘. **이건 셋 다 '원가를 실시간으로 귀속·통제할 수 있게 만드는' 수단이지, 그 자체가 세일즈 포인트(절감)가 아니다.**

1. **인라인 정책 엔진 (실시간 예산 하드 리미트)**
   - 요청이 실행되기 *전에* 프록시/사이드카 계층에서 세션·고객·기능별 예산을 평가해 한도 초과 호출을 차단. 사후 청구(pass-through)가 아니라 실행 전 게이트.
   - 폭주(overnight spike)를 실시간으로 끊고, 차단 사유를 감사 로그로 남긴다 → CTO가 가장 두려워하는 "Surprise Bill" 구조적 예방.
   - SPOF·지연 우려는 사이드카·비동기·바이패스 모드로 방어.

2. **시맨틱 사용량 귀속 (총원가 뷰의 전제)**
   - 모든 호출을 "어떤 사용자 요청 → 어떤 에이전트 스텝 → 어떤 LLM/툴/외부 API"로 태깅. LLM 단가표를 넘어 **비-LLM API 단가 카탈로그**까지 유지해, 흩어진 총원가를 요청 단위로 합산.
   - 결과물: "고객 A 이번 달 AI 원가 ₩X, 요금제 대비 마진 Y%" — 빌링 데이터가 못 주는 실시간 유닛 이코노믹스.

3. **동적 모델 라우팅 (부수 효과로서의 절감)**
   - 쿼리 복잡도에 따라 단순 요청은 저비용 모델로, 복잡 추론은 프론티어 모델로 라우팅. 공개 연구(RouteLLM 등) 기준 품질 저하 없이 토큰 지출을 상당폭 절감 가능.
   - **위치 주의:** 이건 "덤"이다. 라우팅·절감은 공개 기법이라 해자가 아니며, 메인 메시지(원가 설명)를 흐리지 않도록 "원가를 아니까 자동으로 아낄 수도 있다"는 부수 효과로만 언급.

### Slide 8 — Moat (진짜 진입 장벽)

**헤드라인:** "따라잡히는 건 기능이다. 따라잡을 수 없는 건 데이터다."

**본문 카피:**
- 캐싱·라우팅·하드 리미트는 6개월이면 누구나 복제한다. 그건 기본기지 해자가 아니다. (RouteLLM은 공개 연구, 게이트웨이 예산 기능은 이미 성숙)
- 우리 해자는 **고객 데이터가 쌓일수록 강해지는 두 가지**다:
  1. **유닛 이코노믹스 정확도 격차** — 트래픽이 쌓일수록 폭주 패턴 감지의 오탐/미탐이 줄고, 출시 전 비용 시뮬레이션이 정확해진다. 후발주자는 학습할 트래픽 0에서 시작한다.
  2. **동종 벤치마크 (후행 moat)** — 익명화된 다수 고객 데이터로 "같은 유형 스타트업 대비 요청당 원가 상위 20%"를 제공. 고객 수에 비례해 가치가 커지고, 후발주자는 구조적으로 불가능.
- 그리고 이 원가 뷰가 고객의 **가격 책정·이사회 보고 루틴에 들어가는 순간 전환 비용이 급등**한다 — 빼면 매달 하던 재무 보고가 멈춘다.

**시간축 요약:**

| 단계 | 무기 | 성격 |
|---|---|---|
| 진입 (0~10 고객) | 경로 밖 원가 리포트 + 크레딧 소진 카운트다운(무료 쐐기) | pain 발생 전 설치 |
| 차별화 (10~50) | 실시간 유닛 이코노믹스 + 통합 단가 카탈로그 + 폭주 패턴 감지 | 데이터 축적 시작 |
| Moat (50+) | 시뮬레이션 정확도 격차 + 동종 벤치마크 | 구조적·모방 불가 |

## 3. 3대 필수 반론 방어 (Q&A, 백업 슬라이드)

| 심사위원 질문 | 준비된 답 |
|---|---|
| **"그냥 토큰 사고 말지?"** | 소규모엔 맞습니다. 그래서 우리 고객은 월 $50K+ 쓰는 스케일업 — 도구값이 지출의 1~3%이고, 원가를 몰라 잘못 매긴 가격 하나가 그보다 훨씬 큽니다. |
| **"Meta처럼 직접 만들면?"** | Meta는 엔지니어 수천 명이 있습니다. 우리 고객(50~300명 스케일업)은 없습니다. Meta가 증명한 문제를, 자체 구축이 불가능한 99%에게 가져갑니다. |
| **"OpenAI/Anthropic이 하면?"** | 프로바이더는 자기 지출만 봅니다. 고객 통증은 크로스 벤더 합계이고, 벤더 자기 감사에는 증거 능력이 없습니다(재무·이사회 제출 불가). |
