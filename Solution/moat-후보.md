# Moat 후보 발굴 — 경쟁사에 없고 우리에게 있을 수 있는 것

> ⚠️ 상태: 가설. [../Competitor-Analysis/기능-비교-매트릭스.md](../Competitor-Analysis/기능-비교-매트릭스.md)의 "전원 ❌" 줄에서 도출. 수요는 인터뷰로, 실현 가능성은 프로토타입으로 검증 전.

## 원칙: 기능 갭 ≠ moat

경쟁사에 없는 기능은 **차별화**일 뿐이다. 출시 후 6개월이면 베낄 수 있는 건 moat가 아니다. moat는 세 가지에서만 나온다: **① 시간이 지날수록 쌓이는 데이터, ② 커질수록 강해지는 네트워크 효과, ③ 높아지는 전환 비용.** 아래 후보들을 이 기준으로 평가한다.

## 후보 목록 (매트릭스 "전원 ❌"에서 도출)

### 후보 1. 루프·폭주 패턴 감지 엔진

- **무엇**: LiteLLM식 고정 상한(max_iterations)이 아니라, 정상 세션의 호출 패턴을 학습해 "이 세션은 무한 재시도 루프에 빠졌다"를 실시간 탐지·차단. why-now의 "$47K, 11일간 인지 못함" 사고의 직접 해법.
- **왜 아무도 없나**: 고정 상한은 쉽고 패턴 감지는 데이터가 필요하기 때문. 프리론칭 경쟁사들은 아직 학습할 트래픽이 없다.
- **moat 강도: 중~상 (①데이터 축적)** — 고객 트래픽이 쌓일수록 오탐/미탐이 줄어든다. 후발주자는 데이터 없이 시작해야 한다.
- **검증**: 인터뷰 Q7~Q8에서 실제 폭주 사고의 패턴 수집. 고정 상한으로 못 막았던 사례가 있는지.

### 후보 2. 실시간 유닛 이코노믹스 뷰 (고객 1명당 AI COGS·기능별 마진)

- **무엇**: 빌링 데이터(사후·월 단위)가 아니라 호출 경로 데이터에서 실시간으로 "고객 A의 이번 달 AI 원가 ₩X, 요금제 대비 마진 Y%"를 계산. FinOps는 사후+엔터프라이즈, 게이트웨이 진영은 지출 합계만.
- **moat 강도: 중 (③전환 비용)** — 이 뷰가 가격 책정·이사회 보고에 들어가기 시작하면 빼기 어렵다. 단, 기능 자체는 모방 가능.
- **검증**: 인터뷰 Q5·Q15 + CTO 페르소나의 "투자 실사·가격 책정" 트리거가 실재하는지. **우리 차별화 전체의 최우선 검증 대상.**

### 후보 3. 크레딧 소진 카운트다운 & 실비 전환 시뮬레이션

- **무엇**: "현재 소진 속도면 AWS 크레딧이 11월 14일에 끝나고, 그날부터 월 ₩2,300만 실비가 나간다"를 상시 표시. ICP의 크레딧 소진 트리거를 제품 기능으로 만든 것.
- **왜 아무도 없나**: 글로벌 제품들은 크레딧을 온보딩 마케팅으로만 볼 뿐 관리 대상으로 안 본다. **한국 스타트업 생태계(크레딧 의존도 높음)에 특화된 진입 기능.**
- **moat 강도: 하 (모방 쉬움)** — 하지만 moat가 아니라 **쐐기(wedge)**로서 가치: pain이 시작되기 전에 도입시키는 장치. 무료 기능으로 뿌려 bottom-up 진입.
- **검증**: 인터뷰 Q3. 크레딧 소진일을 지금 즉답할 수 있는 CTO가 몇이나 되는지.

### 후보 4. 서드파티 API 단가 카탈로그 (LLM + 비 LLM 통합 요금표)

- **무엇**: LLM 모델 단가는 물론, 에이전트가 부르는 검색 API·크롤링·서드파티 SaaS의 단가를 유지·갱신하는 카탈로그. 실행 전 비용 예약(얼마를 reserve할지)과 통합 COGS 계산의 전제 조건.
- **왜 아무도 없나**: 지루한 궂은일이고, LLM 게이트웨이들은 LLM 단가표까지만 관리한다.
- **moat 강도: 중~상 (①축적 + 규모의 경제)** — 커뮤니티/고객이 늘수록 카탈로그가 정확·완전해진다. OSS로 공개하면 표준화 네트워크 효과(②)도 가능. 후보 1·2·5의 기반 인프라.
- **검증**: 인터뷰 Q6. 외부 API 비용이 실제로 흩어져서 고통인지.

### 후보 5. 출시 전 비용 시뮬레이션 ("이 기능 켜면 월 얼마?")

- **무엇**: 새 에이전트 기능의 예상 트래픽 × 호출 패턴 × 단가 카탈로그로 출시 전 비용 범위를 추정. 챔피언 페르소나의 "출시 전 비용 질문에 답 못 함" pain의 해법.
- **moat 강도: 중 (①)** — 시뮬레이션 정확도가 실측 데이터 축적에 비례.
- **검증**: 인터뷰에서 이 질문을 실제로 받아봤는지, 지금은 어떻게 답하는지.

### 후보 6. 동종 벤치마크 ("업계 대비 요청당 원가 상위 20%")

- **무엇**: 익명화된 고객 데이터로 "같은 AI CS 에이전트 스타트업 대비 우리 원가 수준" 비교 제공.
- **moat 강도: 상 (①+② 완전체)** — 고객 수에 비례해 벤치마크 가치가 커지고, 후발주자는 구조적으로 불가능. **단, 고객이 충분히 모인 후에만 성립하는 후행 moat** — 초기 차별화로는 쓸 수 없다.
- **검증**: 후순위. 고객 10곳 이후의 로드맵.

## 종합: 시간축 위의 전략 (가설)

| 단계 | 무기 | 성격 |
|---|---|---|
| 진입 (0~10 고객) | 후보 3 (크레딧 카운트다운) 무료 배포 | wedge — pain 발생 전에 설치 |
| 차별화 (10~50) | 후보 2 (유닛 이코노믹스) + 후보 4 (통합 카탈로그) + 후보 1 (패턴 감지) | 기능 갭 → 데이터 축적 시작 |
| Moat (50+) | 후보 1·5의 정확도 격차 + 후보 6 (벤치마크) | 구조적 — 모방 불가 영역 진입 |

**전제되는 리스크**: SatGate·Cycles가 출시를 완료하면 매트릭스의 ⚠️(외부 API 차단, 툴 콜 귀속)가 ✅로 바뀐다 — 그 영역(B군 기능)은 우리도 기본기로 필요하지만 moat는 아니다. moat는 위 표의 재무·인텔리전스 레이어(C군의 전원 ❌ 줄)에서 만든다는 것이 이 문서의 핵심 베팅이다.
