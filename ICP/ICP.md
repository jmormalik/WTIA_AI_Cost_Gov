# ICP: 1차 타겟 — 한국 AI 네이티브 스타트업

> ⚠️ 상태: 검증되지 않은 가설. 인터뷰로 확인된 사실 아님.
>
> 결정 기록 (2026-07-20): 한국 시장 우선, AI 네이티브 스타트업 세그먼트 우선으로 확정. 근거는 [../Problem/문제-정의.md](../Problem/문제-정의.md)와 [../Competitor-Analysis/경쟁사-분석.md](../Competitor-Analysis/경쟁사-분석.md).

## 1. 세그먼트 정의

**AI 네이티브 스타트업** — LLM 기반 에이전트/AI 기능이 제품의 핵심 가치인 한국 스타트업. AI를 "도입"한 회사가 아니라 AI가 곧 제품인 회사.

이 세그먼트를 1차로 고른 이유 (문제 적합성):

- **AI 비용이 곧 원가(COGS)다.** 에이전트가 자율적으로 툴 콜/API를 호출할 때마다 마진이 깎인다. 국내 자료 기준 에이전트 운영 시 LLM API 비용이 운영비의 40~50%까지 차지할 수 있다 — 비용 예측 실패가 사업 모델 문제로 직결된다.
- **의사결정이 빠르다.** 엔터프라이즈처럼 조달·보안 심사 없이 창업자/CTO가 바로 결정한다. 검증 사이클이 짧다.
- **status quo 행동이 실제로 관찰된다.** 게이트웨이·관측성 도구 없이 로그 뒤지기·스크립트 짜기로 버티는 단계의 회사가 많다.

## 2. 프로필 (Firmographics + AI 사용 형태)

| 축 | 기준 (가설) |
|---|---|
| 규모 | 팀 10~100명, 시드~시리즈 B. 엔지니어 5명 이상 (자체 개발 조직 보유) |
| 산업 | 제품에 에이전트를 내장한 B2B/B2C SaaS — AI 검색·리서치, AI 고객응대(CS), AI 업무자동화, 버티컬 에이전트(법률·의료·커머스 등) |
| AI 사용 형태 | **에이전트가 프로덕션에서 고객 트래픽을 받는다** (파일럿·데모 아님). 멀티스텝 툴 콜 / MCP / 외부 API 호출(검색 API, 크롤링, 서드파티 SaaS)이 응답 생성 경로에 포함 |
| AI 지출 | 월 LLM API 지출 수백만 원 이상 (~$5K+), 성장 추세. 또는 **하이퍼스케일러 크레딧으로 지출 중이며 6개월 내 소진 예정** |
| 스택 | 멀티 프로바이더(OpenAI + Anthropic 등) 또는 멀티 모델 사용. LiteLLM/Langfuse류 도구를 이미 쓰거나 검토 중이면 강한 신호 |

**핵심 타이밍 트리거 (가설):** 한국 스타트업 다수가 AWS/Azure/GCP 크레딧(최대 수억 원)으로 LLM 비용을 흡수하고 있다. **크레딧이 소진되어 실비 청구가 시작되는 시점**에 비용 추적·통제 pain이 급증한다 — 이 시점의 회사가 최적 진입 대상.

## 3. 관찰 가능한 시그널 (인터뷰 대상 발굴용)

- 제품 페이지에 "AI 에이전트", "자율 수행", 멀티스텝 작업 완수를 내세움
- 채용공고에 LLM Ops / AI 인프라 / 프롬프트 엔지니어링 포지션
- 기술 블로그에 LLM 비용 최적화·모델 라우팅·캐싱 관련 글
- LiteLLM, Langfuse, Helicone 등 도입 흔적 (기술 블로그, GitHub, 채용공고 스택 명시)
- 하이퍼스케일러 스타트업 프로그램(Bedrock/Azure/GCP 크레딧) 수혜 이력
- 후보 풀 예시 (프로필 부합 여부는 미검증): AI 검색·리서치(라이너 등), 업무 자동화 에이전트(달파 등), AI 네이티브 SaaS 전반. Google for Startups AI First Korea, 하이퍼스케일러 프로그램 배치 리스트가 발굴 소스로 유용

## 4. Anti-ICP (제외 기준)

| 제외 대상 | 이유 |
|---|---|
| 월 AI 지출 100만 원 미만 | 문제 심각도 임계점 미달 — status quo("아무것도 안 함")로 충분한 단계 |
| 단순 챗봇/단일 LLM 호출 제품 (툴 콜 없음) | 우리 문제 정의(자율적 툴 콜의 추적·통제)가 발생하지 않음. 비용이 예측 가능함 |
| 자체 파운데이션 모델 개발사 (Upstage, Trillion Labs 등) | 비용 구조가 API 호출이 아니라 GPU 인프라 — 다른 문제 |
| AI 도입 검토 단계의 전통 기업 | 프로덕션 트래픽이 없어 pain 미발생. (엔터프라이즈 내부 도입은 2차 세그먼트로 보류) |
| SI/외주로 에이전트 구축해주는 에이전시 | 비용 부담 주체가 고객사라 pain의 소유자가 아님 |

## 5. 인터뷰 스크리닝 질문

ICP 부합 여부를 5분 안에 거르는 질문:

1. "에이전트/AI 기능이 지금 실제 고객 트래픽을 받고 있나요?" — 프로덕션 여부
2. "응답을 만들 때 AI가 검색, DB 조회, 외부 API 같은 툴을 스스로 호출하나요? 한 요청에 보통 몇 번이나요?" — 자율 툴 콜 여부·규모
3. "월 LLM/API 비용이 대략 어느 규모이고, 크레딧인가요 실비인가요? 크레딧이면 언제 소진되나요?" — 지출 규모·타이밍 트리거
4. "지난 3개월 안에 AI 비용이 예상보다 튀어서 원인을 찾아본 적 있나요? 어떻게 찾았나요?" — status quo 행동(로그 뒤지기·스크립트) 확인
5. "기능·고객·팀 단위로 AI 비용을 나눠 볼 수 있나요?" — 귀속(attribution) 능력 확인

## 6. 후순위 세그먼트 (보류 기록)

- **(b) 엔터프라이즈 내부 에이전트 도입 기업** — 비용 귀속·감사 요구는 강하나 조달 사이클이 길다. 1차 검증 후 재평가
- **(c) 멀티테넌트 에이전트 플랫폼 운영사** — 테넌트별 예산 격리 니즈가 가장 기술적으로 절실하나 국내 모수가 작음. SatGate가 노리는 타겟과 겹침

## ICP 전환 트리거 — 어떤 데이터가 나오면 세그먼트를 바꾸는가

> 목적: 전환 결정을 감이 아니라 사전에 합의된 기준으로 하기 위함 (2026-07-21 추가). 배경: 시장 산정([../Market/TAM-SAM-SOM.md](../Market/TAM-SAM-SOM.md))에서 한국 스타트업 세그먼트의 SAM이 작게 나왔으나, 검증 0건 상태에서의 전환은 보류하기로 결정. 현재 전략은 "스타트업에서 검증·시작 → 고객과 함께 업마켓" 시퀀스.

**전환을 검토하는 조건 (인터뷰 5~10건 축적 후 판정):**

| # | 트리거 | 측정 방법 | 전환 방향 |
|---|---|---|---|
| 1 | 스타트업이 "pain은 있는데 돈은 안 낸다"로 일관 | Q7~Q10에서 pain 확인됐는데 Q14(지불 이력)·Q16(커밋먼트)이 5건 이상 연속 빈손 | 미드마켓/스케일업 (시리즈 C+, 100~500명, ACV $30~50K) |
| 2 | 스크리닝 통과율이 예상(15~25%)을 크게 하회 | Q1~Q3 탈락률 추적 — 후보 10곳 중 1곳 이하만 통과하면 모수 가설 붕괴 | ICP 기준 완화 또는 세그먼트 재정의 |
| 3 | 엔터프라이즈 신호가 반복 등장 | 인터뷰·아웃리치 응답에서 감사, 컴플라이언스, 부서별 귀속, 조달 요구가 3회 이상 자발적으로 언급 | 미드마켓 경유 엔터프라이즈 |
| 4 | 크레딧 트리거 가설 기각 | Q3에서 크레딧 소진 시점과 pain 강도가 무상관으로 판명 | 전환 아닌 wedge 전략(moat 후보 3) 폐기 — ICP는 유지 |
| 5 | 챔피언 경로 부재 | 인터뷰에서 CTO/AI 리드 분리 구도가 확인 안 되고 도입 결정이 전부 top-down | 페르소나 통합 + 영업 모델 수정 — ICP는 유지 |

**전환하지 않는 조건 (노이즈로 취급):**

- 인터뷰 1~2건의 부정 신호 — 표본 부족. 최소 5건 후 판정
- "엔터프라이즈가 돈이 더 많다"는 일반론 — 데이터 아님. 위 트리거 3처럼 자발적 언급이 반복될 때만 인정
- 경쟁사(SatGate 등)의 엔터프라이즈 행보 — 그들의 선택이지 우리 데이터가 아님

**전환 시 잃는 것 (결정 전 재확인용):** bottom-up 챔피언 진입 경로, 셀프서브 모델, 크레딧 카운트다운 wedge(스타트업 전용), 검증 속도(엔터프라이즈 조달 6~18개월). 전환은 이 넷을 버릴 만큼 트리거가 강할 때만.

## 검증 계획

- 3장의 시그널로 후보 기업 리스트업 → 5장의 스크리닝 질문으로 인터뷰 진행 → 결과는 `Interview/YYYY-MM-DD-대상.md`로 기록
- 인터뷰에서 확인할 핵심: 2장의 프로필 기준(특히 지출 임계점·크레딧 트리거)이 실제 pain과 상관 있는지, [../Competitor-Analysis/경쟁사-분석.md](../Competitor-Analysis/경쟁사-분석.md)의 하위 가설(사후 관측 vs 실행 전 통제에 대한 지불 의사)
- ICP가 검증되면 이 프로필 안의 의사결정자(창업자/CTO/AI 리드)를 `Buyer-Persona/`로 전개
