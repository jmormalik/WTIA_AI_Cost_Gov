# ICP: 유료 타겟 = 중간 시장/스케일업 · 한국 AI 스타트업 = 무료 디자인 파트너

> ⚠️ 상태: 검증되지 않은 가설. 인터뷰로 확인된 사실 아님 (정식 인터뷰 0건).
>
> 결정 기록:
> - 2026-07-20: 한국 AI 네이티브 스타트업을 1차 타겟으로 확정 (구버전).
> - **2026-07-22: ICP 재설정** — 유료 타겟을 소규모 스타트업 → **중간 시장/스케일업**으로 전환. 근거는 [../Solution/포지셔닝-재설정.md](../Solution/포지셔닝-재설정.md) (죽음의 계곡: 소규모는 ROI 음수, 대기업은 자체구축). 한국 AI 스타트업은 유료 ICP에서 **무료 디자인 파트너**로 역할 변경. 이 전환은 인터뷰 데이터가 아니라 반론 논리(Meta 사례 + ROI 계산)에 근거한 전략 재설정이므로, 인터뷰에서 역방향 신호가 나오면 §7 트리거로 재검토한다.

---

## 1. 유료 타겟 — 중간 시장/스케일업 (글로벌, 미국 중심)

**정의:** AI가 제품 핵심인 스케일업 중, **Meta처럼 자체 게이트웨이를 만들 엔지니어 조직은 없는데, 스타트업보다 지출·감사 압박은 큰 회사.** ← 핵심 판별 기준

이 세그먼트를 유료 타겟으로 고른 이유:

- **월 AI 지출 $50K+에서 도구값이 지출의 1~3%** → ROI 논쟁이 사라진다 (소규모는 10~20%라 "차라리 토큰 사고 만다").
- **재무·이사회·감사 압박이 실재한다.** 유닛 이코노믹스 보고, 가격 책정, 투자 실사에 AI 원가 귀속(COGS)이 필수 — 절감이 아니라 **설명이 상시 니즈**.
- **전담 인프라 플랫폼팀을 꾸릴 만큼 크지 않다** → 직접 만들 수 없어 사야 한다.

### 프로필 (Firmographics + AI 사용 형태)

| 축 | 기준 (가설) |
|---|---|
| 규모 | 팀 50~300명, 시리즈 A~C(대략 B~D까지 허용 — 판별은 규모·자체구축 능력으로) |
| 월 AI 지출 | **$50K~$500K**, 실비 청구 단계 (크레딧 소진 완료 또는 크레딧으로 못 가리는 규모) |
| AI 사용 형태 | **에이전트가 프로덕션에서 고객 트래픽을 받는다.** 멀티스텝 툴 콜 / MCP / 외부 API 호출(검색·크롤링·서드파티 SaaS)이 응답 생성 경로에 포함 |
| 스택 | **서드파티 모델(OpenAI·Anthropic 등) 위에 구축** — 자체 파운데이션 모델 개발사 아님. 멀티 프로바이더면 강한 신호 |
| 조직 신호 | 재무/RevOps 기능이 생겼고 이사회에 유닛 이코노믹스를 보고해야 함. 고객사/테넌트 단위 과금 모델이면 원가 귀속 pain 구조적 |

### 관찰 가능한 시그널 (후보 발굴용)

- 기술 블로그에 LLM 비용 최적화·모델 라우팅·캐싱·토큰 원가 관련 글
- 채용공고에 LLM Ops / AI 인프라 / 플랫폼 엔지니어 포지션 (단 "AI Gateway 팀" 채용이면 자체구축 경로 — 경계 신호)
- 사용량 기반 AI 과금(per-resolution, per-seat+usage 등) 가격 모델 — 고객당 COGS 인식이 이미 있다는 뜻
- 창업자·CTO의 공개 발언에 토큰 비용·마진 언급 (예: Harvey "계약 검토 1건에 토큰 $20K")
- LiteLLM/Langfuse/Helicone류 도입 흔적 — 관측까지는 했는데 귀속·감사는 못 하는 단계면 최적

## 2. 무료 디자인 파트너 — 한국 AI 네이티브 스타트업

**역할:** 유료 시장이 아니다. ① 가설 0·가설 1 검증(툴/외부API 비중, COGS 상시 니즈) ② 샘플 로그 확보(Cost Autopsy) ③ 덱 트랙션 대용("디자인 파트너 N곳") — 접근성·의사결정 속도가 빠른 검증 채널. 피치 시장 스토리 = **"한국 = 쐐기, 시장 = 글로벌 에이전트 인프라."**

**정의:** LLM 기반 에이전트/AI 기능이 제품의 핵심 가치인 한국 스타트업. AI를 "도입"한 회사가 아니라 AI가 곧 제품인 회사.

| 축 | 기준 (가설) |
|---|---|
| 규모 | 팀 10~100명, 시드~시리즈 B. 엔지니어 5명 이상 (자체 개발 조직 보유) |
| 산업 | 제품에 에이전트를 내장한 B2B/B2C SaaS — AI 검색·리서치, AI CS, 업무자동화, 버티컬 에이전트(법률·의료·커머스 등) |
| AI 사용 형태 | 에이전트가 프로덕션에서 고객 트래픽을 받는다 (파일럿·데모 아님). 멀티스텝 툴 콜이 응답 경로에 포함 |
| AI 지출 | 월 ~$5K+ 또는 **하이퍼스케일러 크레딧 지출 중, 6개월 내 소진 예정** |
| 스택 | 멀티 프로바이더 또는 멀티 모델. LiteLLM/Langfuse류를 쓰거나 검토 중이면 강한 신호 |

**타이밍 트리거 (가설):** 크레딧이 소진되어 실비 청구가 시작되는 시점에 비용 추적·통제 pain이 급증 — 이 시점의 회사가 최적 접촉 대상.

**발굴 시그널:** 제품 페이지의 "AI 에이전트"·자율 수행 문구, LLM Ops 채용공고, 비용 최적화 기술 블로그, 하이퍼스케일러 크레딧 프로그램(Bedrock/Azure/GCP) 수혜 이력. 후보 풀: AIIA AI 에이전트 분야, Google for Startups AI First Korea, AWS 유니콘데이 참가사 (상세: [../Interview/인터뷰-후보-리스트.md](../Interview/인터뷰-후보-리스트.md)).

> ⚠️ 미해결 (팀 확인 대기): 인터뷰 스크리닝 필수 조건(매출 주원천 = 유료/B2B)이 "한국 스타트업 = 무료 디자인 파트너" 전략과 충돌하는지 — 유료 검증용 표본과 디자인 파트너 후보를 같은 파이프라인에서 다르게 태깅할지 결정 필요.

## 3. Anti-ICP (제외 기준)

| 제외 대상 | 이유 |
|---|---|
| 월 AI 지출 $5K 미만 소규모 스타트업 (유료 타겟으로서) | 도구값이 지출의 10~20% → ROI 음수, "차라리 토큰 사고 만다"가 합리적. 디자인 파트너로는 가능 |
| 대기업 (엔지니어 조직 수천 명) | Meta·Uber처럼 **직접 만든다** — 구매 안 함 |
| 단순 챗봇/단일 LLM 호출 제품 (툴 콜 없음) | 자율적 툴 콜의 추적·통제 문제가 발생하지 않음. 비용이 예측 가능 |
| 자체 파운데이션 모델 개발사 | 비용 구조가 API 호출이 아니라 GPU 인프라 — 다른 문제 |
| AI 도입 검토 단계의 전통 기업 (AX 전환 초기) | 프로덕션 트래픽이 없어 pain 미발생. 지출도 임계점 미달 |
| SI/외주로 에이전트 구축해주는 에이전시 | 비용 부담 주체가 고객사라 pain의 소유자가 아님 |
| 개발 도구/코딩 에이전트 (Cursor, Cognition 등) | 인접 시장 — why-now 구분 대상. 대조군으로만 활용 |

## 4. 의사결정자 (Buyer Persona 연결)

- 1차 접촉: CTO/기술 공동창업자 ([../Buyer-Persona/01-CTO-공동창업자.md](../Buyer-Persona/01-CTO-공동창업자.md)), AI 리드/플랫폼 리드 ([../Buyer-Persona/02-AI-리드-챔피언.md](../Buyer-Persona/02-AI-리드-챔피언.md))
- 중간 시장에서는 **CFO/재무가 경제적 이해관계자**로 뒤에 붙는 구도 (COGS 보고·이사회 압박이 구매 트리거) — 페르소나 문서에 반영 필요 여부는 인터뷰 후 판단

## 5. 인터뷰 스크리닝 질문

ICP 부합 여부를 5분 안에 거르는 질문 (상세 문항은 [../Interview/인터뷰-가이드-Mom-Test.md](../Interview/인터뷰-가이드-Mom-Test.md) §4-2 가설 0·1 문항 참조):

1. "에이전트/AI 기능이 지금 실제 고객 트래픽을 받고 있나요?" — 프로덕션 여부
2. "응답을 만들 때 AI가 검색, DB 조회, 외부 API 같은 툴을 스스로 호출하나요? 한 요청에 보통 몇 번이나요?" — 자율 툴 콜 여부·규모
3. "월 LLM/API 비용이 대략 어느 규모이고, 크레딧인가요 실비인가요?" — 지출 규모 (유료 타겟 판별: $50K+ / 디자인 파트너: $5K+ 또는 크레딧 소진 임박)
4. "고객·기능·팀 단위로 AI 원가를 나눠서 보고해야 할 일이 있나요? 누가 그 숫자를 소비하나요?" — **COGS 귀속·설명 니즈 (가설 1)**
5. "지난 3개월 안에 AI 비용이 예상보다 튀어서 원인을 찾아본 적 있나요? 어떻게 찾았나요?" — status quo 행동 확인
6. "그런 분석을 직접 만들 팀이 있나요, 아니면 사서 쓰는 걸 검토해봤나요?" — **구매 vs 자체구축 임계점**

## 6. 후순위 세그먼트 (보류 기록)

- **(b) 엔터프라이즈 내부 에이전트 도입 기업** — 비용 귀속·감사 요구는 강하나 조달 사이클이 길다. 중간 시장 검증 후 업마켓 경로로 재평가
- **(c) 멀티테넌트 에이전트 플랫폼 운영사** — 테넌트별 예산 격리 니즈가 기술적으로 가장 절실. SatGate가 노리는 타겟과 겹침. 중간 시장 안의 서브세그먼트로 흡수 가능성

## 7. ICP 전환 트리거 — 어떤 데이터가 나오면 다시 바꾸는가

> 목적: 전환 결정을 감이 아니라 사전에 합의된 기준으로 하기 위함. 2026-07-21 최초 작성(스타트업→미드마켓 트리거), **2026-07-22 재설정으로 트리거 1이 데이터 축적 전에 논리 근거로 선제 발동됨** — 따라서 신 트리거는 역방향(미드마켓 가설이 틀렸을 때)을 감시한다.

**전환/수정을 검토하는 조건 (인터뷰 5~10건 축적 후 판정):**

| # | 트리거 | 측정 방법 | 대응 방향 |
|---|---|---|---|
| 1 | 중간 시장도 "직접 만든다"로 일관 | "구매 vs 자체구축" 질문에서 5건 이상 연속 자체구축(또는 OSS 조합으로 충분) 응답 | 죽음의 계곡이 중간 시장까지 확장 — 포지셔닝 자체 재검토 (경로 밖 리포트 단독 상품화 등) |
| 2 | COGS 귀속이 상시 니즈가 아님 (가설 1 기각) | 가설 1 판정식: 5건+ 생존 / 3건− 기각 (인터뷰 가이드 참조) | 가치 프레임 재설계 — Semantic ROI Kill-switch 등 대안 전면화 |
| 3 | 중간 시장 접근 실패 (응답률 붕괴) | 미국 중간 시장 아웃리치 10곳+ 무응답, 한국 디자인 파트너만 응답 | 검증 채널을 한국에 집중, 미국은 2차 자료·자문 경유로 우회 |
| 4 | 크레딧 트리거 가설 기각 | 크레딧 소진 시점과 pain 강도가 무상관으로 판명 | 디자인 파트너 발굴 wedge만 폐기 — ICP는 유지 |
| 5 | 챔피언 경로 부재 | CTO/AI 리드 분리 구도 확인 안 되고 도입 결정이 전부 top-down | 페르소나 통합 + 영업 모델 수정 — ICP는 유지 |

**전환하지 않는 조건 (노이즈로 취급):**

- 인터뷰 1~2건의 부정 신호 — 표본 부족. 최소 5건 후 판정
- "엔터프라이즈가 돈이 더 많다"는 일반론 — 데이터 아님
- 경쟁사(SatGate 등)의 행보 — 그들의 선택이지 우리 데이터가 아님

## 8. 검증 계획

- 유료 타겟(미국 중간 시장): 시그널 기반 후보 리스트업 → 링크드인 콜드 아웃리치 → 스크리닝 → 인터뷰. **핵심 확인:** 구매 vs 자체구축 임계점, COGS 귀속의 상시성(가설 1), 툴/외부API 비중(가설 0), ACV 지불 의사
- 디자인 파트너(한국): 아웃리치 진행 중 ([../Interview/아웃리치-파이프라인.md](../Interview/아웃리치-파이프라인.md)). **핵심 확인:** 가설 0·1 + 샘플 로그 제공 의향(행동 지표)
