# Buyer Persona 2: AI 리드 / 백엔드 테크리드 — 챔피언 (Champion / Daily User)

> ⚠️ 상태: 검증되지 않은 가설. [../ICP/ICP.md](../ICP/ICP.md)에서 유도. 인터뷰로 확인된 사실 아님.
>
> 관계: 이 페르소나가 도구를 발견·도입 제안하고([01-CTO-공동창업자.md](01-CTO-공동창업자.md))가 결제를 승인하는 구도를 가정. 오픈소스/무료 티어라면 이 페르소나가 결재 없이 혼자 도입할 수도 있다 — bottom-up 진입 경로.

## 프로필

| 항목 | 내용 (가설) |
|---|---|
| 역할 | AI 엔지니어링 리드, 백엔드 테크리드, 또는 "LLM 파이프라인을 만든 그 사람". 팀 10~100명 스타트업에서 에이전트 오케스트레이션 코드를 직접 소유 |
| 배경 | 백엔드 출신으로 LLM을 프로덕션에 올려본 1~3년차 경험. LiteLLM·Langfuse류 도구를 이미 알거나 써봤다 |
| 권한 | 스택 선택은 사실상 본인 재량. 유료 전환 시에만 CTO 승인 필요 |
| 일상 | 에이전트 품질·레이턴시·비용 사이의 트레이드오프를 매일 조정. 비용이 튀면 청구서를 들고 오는 건 CTO지만, **로그를 뒤지는 건 이 사람** |

## 목표 (성공 기준)

- 에이전트가 품질을 유지하면서 예측 가능하게 동작하는 것 — 루프·리트라이 폭주 없이
- 비용 질문("이번 달 왜 이래?")에 로그 뒤지지 않고 바로 답할 수 있는 것
- 비용 집계 스크립트 같은 **제품 아닌 일**에서 벗어나는 것

## Pain Points

1. **사고의 최전선.** 비용이 튀면 원인 추적은 이 사람 몫 — 요청 로그, 프로바이더 대시보드, 자체 스크립트를 오가며 반나절~며칠 소모 (status quo의 "로그 뒤지기" 당사자).
2. **가시성의 사각지대.** LLM 토큰은 그나마 보이는데, 에이전트가 부르는 외부 API(검색, 크롤링, 서드파티 SaaS) 비용은 각 서비스 대시보드에 흩어져 있어 합산이 안 된다 (하위 가설 2).
3. **통제 수단의 딜레마.** 프로바이더 하드 캡은 서비스를 통째로 죽이고, 알림은 이미 늦다. 세밀한 통제(세션·기능·고객 단위 상한)를 만들려면 직접 짜야 한다.
4. **재발 방지 압박.** 사고 한 번 나면 "다시는 안 나게 하라"는 요구를 받지만, 모델이 호출을 결정하는 구조에서 완전한 재발 방지를 약속할 수 없다.

## 도입 트리거 (가설 — 인터뷰로 검증)

- 비용 사고 후 포스트모템에서 "재발 방지 대책" 액션 아이템을 받았을 때
- 비용 집계 스크립트가 프로바이더 추가·모델 변경으로 또 깨졌을 때
- 새 에이전트 기능 출시 전 "이거 비용 얼마나 나올지" 질문에 답을 못 했을 때

## 도입 장벽 (예상 반론)

- "호출 경로에 뭘 끼우는 건 무섭다" — 레이턴시·단일 장애점(SPOF) 우려. 사이드카/비동기 모드, 장애 시 bypass 보장이 필요
- "Langfuse 이미 쓰는데 뭐가 달라?" — 관측(사후)과 통제(실행 전)의 차이를 체감시켜야 함
- "직접 짜면 된다" — NIH 성향. 다만 유지보수 피로가 쌓인 상태라면 오히려 전환 신호

## 정보 채널

GitHub trending, Hacker News, 기술 블로그, 국내 개발자 커뮤니티(GeekNews 등), 사내 슬랙에 공유되는 도구 소문. **오픈소스 여부와 문서 품질이 첫인상을 결정.**

## 메시지 앵글 (가설)

"비용 관리"가 아니라 **"에이전트 폭주를 코드 한 줄로 막고, 비용 질문에 10초 만에 답한다"(엔지니어링 문제 해결)**. 이 페르소나에게 비용은 재무 문제가 아니라 관측성·안정성 문제다.

## 두 페르소나의 관계 (영업 가설)

- **진입**: 챔피언(이 페르소나)이 오픈소스/무료 티어로 먼저 써보고 → 사고 또는 크레딧 소진 시점에 CTO에게 유료 전환 제안
- **메시지 분리**: 챔피언에게는 엔지니어링 언어(폭주 방지·가시성), CTO에게는 사업 언어(유닛 이코노믹스·런웨이)
- 인터뷰에서 확인할 것: 실제로 이 구도인지, 아니면 이 규모에선 CTO가 직접 다 하는지 (그 경우 페르소나 통합)
